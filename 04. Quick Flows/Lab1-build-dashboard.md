# 자연어로 대시보드 구축하기

Quick Sight는 대시보드 생성을 매우 간단하게 만들어줍니다. 기술적 스킬이나 교육이 필요하지 않습니다. 복잡한 도구를 배우는 대신, 데이터에 대해 일반 언어로 질문하기만 하면 Quick Sight가 자동으로 적절한 시각화를 생성합니다. 이 섹션에서는 전략적 질문을 통해 시민 서비스 대시보드를 구축하는 방법을 시연합니다.

## Quick Sight 데이터셋과 토픽 이해하기

대시보드를 구축하고 질문하기 전에 다음이 필요합니다:

1. **데이터셋 생성**: 데이터 파일을 Quick Sight에 업로드
2. **토픽 생성**: Quick Sight가 데이터를 이해하고 쿼리할 수 있도록 데이터셋 구성

**데이터셋(Dataset)**은 원시 데이터 소스(CSV 파일 등)이고, **토픽(Topic)**은 다음을 정의하는 큐레이션된 컬렉션입니다:

- **데이터 필드**: 사용 가능한 지표와 차원 (서비스 유형, 부서, 날짜 등)
- **비즈니스 용어**: 조직에서 사용하는 일반적인 표현 (기술적 필드명 대신 "해결 시간" 등)
- **관계**: 서로 다른 데이터 요소가 연결되는 방식
- **계산**: 평균, 합계, 백분율 같은 사전 정의된 지표

## 단계 1: 데이터셋 생성

먼저 시민 서비스 데이터를 Quick Sight에 업로드합니다:

1. 왼쪽 내비게이션에서 **Datasets**로 이동합니다.
2. **Create dataset**을 클릭합니다.
3. **Upload file**을 선택합니다.
4. 워크샵 자료에서 `citizen_services_data.csv` 파일을 선택합니다.
5. **Edit settings and prepare data**를 클릭합니다.
6. 데이터 미리보기에서 필드가 올바르게 감지되었는지 확인합니다:
   - `request_id`, `request_type`, `category`, `status`, `priority`
   - `submitted_date`, `resolved_date`, `resolution_days`, `department`
   - `neighborhood`, `satisfaction_score`, `budget_code`
7. 상단 바에서 `citizen_services_data.csv` 옆의 연필 아이콘을 클릭합니다. 데이터셋 이름을 `Citizen Services Data`로 변경하고 체크 마크 아이콘을 클릭하여 확인합니다.
8. **Save & publish**를 클릭합니다.

Quick Sight가 파일을 처리하고 데이터셋을 생성합니다. 일반적으로 30~60초가 소요됩니다.

> ℹ️ **수행 결과**: 원시 데이터를 Quick Sight에 업로드하여 분석과 시각화에 사용할 수 있게 되었습니다.

## 단계 2: 데이터셋에서 토픽 생성

이제 이 데이터를 자연어로 쿼리할 수 있는 토픽을 생성합니다. 이는 자연어를 이용한 임시 분석 모듈에서 사용됩니다.

1. 좌측 상단의 **Explore**를 클릭하여 왼쪽 내비게이션 메뉴를 엽니다. **Topics**로 이동합니다.
2. **Create topic**을 클릭합니다.
3. Topic name에 `Citizen Services Operations`를 입력합니다.
4. **Continue**를 클릭합니다.
5. 이전 단계에서 생성한 **Citizen Services Data** 데이터셋을 선택합니다.

Quick Sight가 데이터를 분석하고 자동으로 다음을 수행합니다:

- 주요 필드와 유형 식별
- 비즈니스 친화적 이름 제안
- 데이터 요소 간 관계 감지
- 초기 계산 생성

이 프로세스는 1~2분이 소요됩니다. 그동안 단계 3으로 진행할 수 있습니다.

> ℹ️ **수행 결과**: Quick Sight가 데이터를 인덱싱하고 자연어 인터페이스를 생성하여, 쿼리를 작성하는 대신 일반 영어로 질문할 수 있게 되었습니다.

> ⚠️ **중요 - 사용자 권한**: 토픽 생성 시 오류가 발생하거나 권한이 충분하지 않다는 메시지가 표시되면 Quick Sight 사용자 역할을 업데이트해야 할 수 있습니다. Quick Sight에서 토픽을 생성하고 관리하려면 **Admin Pro** 역할이 필요합니다. 위의 1-5단계를 완료할 수 있었다면 단계 3으로 건너뛸 수 있습니다.
>
> 1. 우측 상단의 프로필 아이콘을 클릭합니다.
> 2. **Manage Quick**을 선택합니다.
> 3. 왼쪽 패널에서 **Manage users**로 이동합니다.
> 4. 목록에서 사용자 이름을 찾습니다.
> 5. 역할이 아직 설정되지 않은 경우 **Admin Pro**로 변경합니다.
> 6. **Confirm**을 클릭하고 위의 1-5단계를 반복합니다.

## 단계 3: 첫 번째 시각화 생성

이제 질문을 통해 시각화를 구축할 준비가 되었습니다. 각 질문은 대시보드에 추가할 수 있는 시각화를 생성합니다.

1. 왼쪽 내비게이션에서 **Analyses**로 이동합니다.
2. **Create analysis**를 클릭합니다.
3. 단계 1에서 생성한 **Citizen Services Data** 데이터셋을 선택합니다.
4. **Select**를 클릭합니다.
5. New sheet 팝업 창에서 **Create**를 클릭합니다. 첫 번째 시각화를 구축할 새 시트가 생성됩니다.
6. Visuals 아래의 **Build** 버튼을 클릭합니다.
7. 우측 패널에 다음과 같은 질문을 입력합니다: `What are the top service request types resolved last year?`
8. **Build**를 클릭합니다. Quick Sight가 질문을 기반으로 자동으로 시각화를 생성합니다. 일반적으로 30~60초가 소요됩니다.
9. **Add to Analysis**를 클릭하여 시각화를 대시보드에 추가합니다.

**결과:**
- Quick Sight가 질문을 해석합니다.
- 적절한 시각화 유형(막대 차트)을 자동으로 선택합니다.
- 가장 일반적인 서비스 유형을 보여주는 결과가 몇 초 내에 나타납니다.

**표시 내용:**
- 요청량별 서비스 유형 순위를 보여주는 막대 차트
- 각 유형의 정확한 수치
- 자동 서식 및 라벨링

> 💡 **전략적 인사이트**: 시민 수요가 어디에 집중되어 있는지 즉시 보여주어 리소스 배분과 예산 계획의 우선순위를 정하는 데 도움이 됩니다.

## 단계 4: 성과 지표 시각화

서비스가 얼마나 빨리 해결되는지 이해해 봅시다. 단계 3과 동일한 과정을 반복하고 다른 질문을 합니다:

```
Show me average resolution time by request type
```

**결과:**
- Quick Sight가 해결 시간을 비교하는 막대 차트를 생성합니다.
- 서비스가 자동으로 해결 시간 순으로 정렬됩니다.
- 어떤 서비스가 완료하는 데 가장 오래 걸리는지 즉시 확인할 수 있습니다.

> 💡 **전략적 인사이트**: 성과 격차를 드러냅니다. 해결 시간이 긴 서비스는 프로세스 개선이나 추가 리소스가 필요할 수 있습니다.

## 단계 5: 지역별 패턴 시각화

수요가 어디서 오는지 이해하면 리소스 배치에 도움이 됩니다. 단계 3과 동일한 과정을 반복하고 다른 질문을 합니다:

```
Which neighborhoods have the most service requests?
```

**결과:**
- Quick Sight가 요청량별 지역 순위를 보여주는 시각화를 생성합니다.
- 어떤 지역의 수요가 가장 높은지 확인할 수 있습니다.
- 시각화가 쉬운 해석을 위해 자동으로 포맷됩니다.

> 💡 **전략적 인사이트**: 지역별 패턴은 형평성 문제, 인프라 필요성, 최적의 리소스 배치 위치를 드러냅니다.

## 단계 6: 시민 만족도 추적

시민 경험을 이해하기 위한 품질 지표를 추가합니다. 단계 3과 동일한 과정을 반복하고 다른 질문을 합니다:

```
What is the average satisfaction score by service type?
```

**결과:**
- Quick Sight가 만족도 등급(1~5 척도)을 보여주는 시각화를 생성합니다.
- 서비스가 만족도 수준별로 순위가 매겨집니다.
- 어떤 서비스에 품질 개선이 필요한지 식별할 수 있습니다.

> 💡 **전략적 인사이트**: 만족도 점수는 서비스 품질을 측정하고 프로세스 개선이 효과가 있는지 검증하는 데 도움이 됩니다.

이 단계들을 완료하면 아래와 유사한 대시보드가 생성되어야 합니다.

## 수행 결과 이해하기

전통적인 BI 도구에서 이러한 시각화를 생성하려면 다음이 필요합니다:

- 복잡한 소프트웨어 인터페이스 학습
- 데이터 구조와 관계 이해
- 차트 유형과 구성 수동 선택
- 쿼리 또는 수식 작성
- 레이아웃 설계 및 서식 지정
- **소요 시간**: 수 시간 또는 수 일, 종종 IT 지원 필요

Quick Sight의 자연어 접근 방식:

- 네 가지 간단한 질문을 했습니다.
- Quick Sight가 적절한 시각화를 자동으로 생성했습니다.
- **소요 시간**: 5~10분, 기술적 스킬 불필요

## 핵심 요점

자연어로 대시보드를 구축하면 다음이 가능합니다:

- **즉시 인사이트 생성**: 수 시간이 아닌 수 초 만에 시각화 구축
- **기술적 스킬 불필요**: 일반 언어로 질문
- **빠른 반복**: 시각화를 쉽게 추가, 제거, 수정
- **전략에 집중**: 도구 구축이 아닌 인사이트 해석에 시간 투자
- **의사결정 역량 강화**: IT를 기다리지 않고 데이터에 직접 접근

이 접근 방식은 데이터 분석을 민주화하여, 인사이트가 필요하지만 복잡한 BI 도구를 배울 시간이 없는 경영진에게 접근 가능하게 만듭니다.
